# ツール一覧画面

**パス:** `/`

**種別:** 要認証（未認証の場合は [`/login`](./login.md) へリダイレクト）

**使用 API:** なし（Phase 2 時点）

> 認証チェックとヘッダー表示は `AppLayout` が担当します。ツール定義は `src/lib/tools.ts` の `TOOLS` を参照します。

## 表示情報

| 項目             | 取得元                |
| ---------------- | --------------------- |
| ツール名         | `TOOLS[].name`        |
| ツール説明       | `TOOLS[].description` |
| ツール遷移先パス | `TOOLS[].path`        |

## UI 要素

- 共通ヘッダー（`Tools` 表示、ログアウトボタン）
- ツールカード一覧（`TOOLS` が 1 件以上の場合）
- 空状態表示（`TOOLS` が 0 件の場合「ツールはまだありません」）
- 認証確認中のローディング表示
- エラーメッセージ表示エリア

## 画面遷移

| 条件                           | 遷移先                                                                                                              |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------- |
| 初期表示時に認証済み           | `/` に留まりツール一覧を表示                                                                                        |
| 初期表示時に未認証             | [`/login`](./login.md) へリダイレクト                                                                               |
| ログアウトボタン押下           | `AUTH_WEB_URL/logout?redirect=<ツール WEB の /login を表す完全 URL>` へ遷移し、完了後 [`/login`](./login.md) に戻る |
| 未定義ルートからのリダイレクト | `/`                                                                                                                 |

## エラーハンドリング

| エラー                     | 表示・処理                                     |
| -------------------------- | ---------------------------------------------- |
| 認証状態判定でエラーが発生 | エラーメッセージを表示し、再ログイン導線を表示 |
| サイレントリフレッシュ失敗 | [`/login`](./login.md) へリダイレクト          |
