# ダウンロード速度測定画面

**パス:** `/tools/download-speed`

**種別:** 要認証（未認証の場合は [`/login`](./login.md) へリダイレクト）

**使用 API:** `GET /tools/download-speed/chunk`

## 画面概要

サーバーから選択サイズ分のバイナリをダウンロードし、クライアント側で経過時間を計測して下り速度（Mbps）を表示します。

## 表示項目

| 項目       | 内容                                                                         |
| ---------- | ---------------------------------------------------------------------------- |
| 計測結果   | 算出した下り速度（Mbps）。完了時に小数第 2 位まで表示する想定                |
| 計測サイズ | ユーザーが選んだチャンクサイズ（10 / 50 / 100 MB）。API の `size_mb` と対応  |
| 所要時間   | ダウンロード開始から完了までの経過時間（ミリ秒または秒の readable 表現）     |
| 計測状態   | 待機（`idle`）／計測中（`measuring`）／完了（`done`）／エラー（`error`）など |

## サイズ選択 UI

- 10 MB / 50 MB / 100 MB のいずれかをボタンなどで選択する
- 計測待機中（例: `idle`）のみサイズを変更できる（計測中は固定）

## データ取得フロー

1. ユーザーがサイズ（10 / 50 / 100 MB）を選ぶ（初期値は 10 MB）
2. 「計測開始」を押す
3. `AbortController` と 30 秒のタイムアウト用タイマーを用意する
4. `GET /tools/download-speed/chunk?size_mb=<選択値>` を `Authorization: Bearer` 付きで呼び出し、レスポンスボディをバイナリとして受け取る
5. 受信完了時刻と開始時刻から経過時間を求め、Mbps を算出して結果エリアに表示する
6. 計測完了・中止・タイムアウトのいずれでもタイマーと `AbortController` をクリーンアップする

## 計測中止

- 計測中のみ「計測中止」ボタンを表示する
- 押下時に `AbortController.abort()` を呼び、フェッチを中断する
- `AbortError` をユーザー向けに「計測を中止しました」などのメッセージとして表示する

## タイムアウト

- 計測開始から 30 秒を超えた場合はエラー扱いとし、`abort()` でリクエストを打ち切る
- 画面上に「計測がタイムアウトしました（30秒）」などと表示する

## エラーハンドリング

| 状況                                 | 表示・処理                                                                         |
| ------------------------------------ | ---------------------------------------------------------------------------------- |
| 計測中止（ユーザー操作）             | エラー状態とし、「計測を中止しました」を表示。再試行で再度計測できるようにする想定 |
| タイムアウト（30 秒）                | エラー状態とし、タイムアウトメッセージを表示                                       |
| API が 401 を返した場合              | 認証エラーとして既存の認証フロー（サイレントリフレッシュ等）に委ねる想定           |
| その他のネットワーク・サーバーエラー | エラーメッセージを表示し、再試行ボタンで再度 `GET` を試せるようにする想定          |
