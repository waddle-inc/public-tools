# ログイン画面

**パス:** `/login`

**種別:** 公開（認証不要）

**使用 API:** なし

> `AUTH_WEB_URL` は環境変数等で与える認証 WEB（フロントエンド）のベース URL を指します。`<コールバック URL>` はツールサービス WEB（フロントエンド）の SSO コールバック（例: `/auth/callback`）への絶対 URL を URL エンコードして付与します。

## クエリパラメータ

| パラメータ | 必須 | 説明                                                                                 |
| ---------- | ---- | ------------------------------------------------------------------------------------ |
| `redirect` | —    | ログイン成功後に遷移する元のパス（例: `/tools/browser-info`）。省略時は `/` を使用。 |

## UI 要素

- 「ログインする」リンク（`href` = `AUTH_WEB_URL/login?redirect=<コールバック URL>`）

## 動作仕様

- `redirect` クエリパラメータが存在する場合、その値を `sessionStorage` の `auth_redirect` キーに保存してから認証 WEB へ遷移する
- `sessionStorage` への保存はボタン押下時（認証 WEB へのリダイレクト直前）に行う

## 画面遷移

| 条件                                                                | 遷移先                                                 |
| ------------------------------------------------------------------- | ------------------------------------------------------ |
| 「ログインする」ボタン押下                                          | 認証 WEB（フロントエンド）のログイン画面へ遷移         |
| 認証 WEB のログアウト後（`redirect` に当画面の URL を付与した場合） | 当画面（`/login`）を表示（ボタン経由と同様に利用可能） |
| 認証済みでアクセス                                                  | [`/`](./tool-list.md) へリダイレクト                   |
