# 文章要約画面

**パス:** `/tools/summarize`

**種別:** 要認証（未認証の場合は [`/login`](./login.md) へリダイレクト）

**使用 API:** `POST /tools/summarize`

## 画面概要

ユーザーが入力したテキストを、選択した長さ指定に応じて Gemini API で要約し、結果を表示します。

## 表示項目

| 項目         | 内容                                                                   |
| ------------ | ---------------------------------------------------------------------- |
| 入力テキスト | 要約対象。テキストエリアで編集する                                     |
| 要約の長さ   | 短め（`short`）／普通（`medium`）／詳しく（`long`）から選択            |
| 要約結果     | API の `summary` を表示する                                            |
| 状態         | 待機（`idle`）／取得中（`loading`）／完了（`done`）／エラー（`error`） |

## 長さ選択 UI

- 「短め」「普通」「詳しく」のいずれかをボタンで選択する（初期値は普通）
- 要約取得中（`loading`）は長さボタンとテキストエリアを無効化する

## データ取得フロー

1. ユーザーがテキストを入力し、要約の長さを選ぶ
2. 「要約を実行」を押す
3. `POST /tools/summarize` に `Authorization: Bearer` と JSON ボディ（`text`, `length`）を送信する
4. 200 の場合は `summary` を結果エリアに表示する
5. 401 の場合は既存の認証フロー（サイレントリフレッシュ等）に委ねる
6. 429（利用上限）や 502 などの失敗はエラーバナーで通知し、ユーザーが再度試せるようにする

## エラーハンドリング

| 状況                                 | 表示・処理                                                                 |
| ------------------------------------ | -------------------------------------------------------------------------- |
| API が 401 を返した場合              | 認証エラーとして既存の認証フローに委ねる想定                               |
| API が 429 を返した場合              | 利用上限として案内し、時間をおいて再試行する想定                           |
| API が 502 を返した場合              | 外部 API 失敗としてエラーメッセージを表示する想定                          |
| その他のネットワーク・サーバーエラー | エラーメッセージを表示し、画面を待機状態に戻して再試行できるようにする想定 |
