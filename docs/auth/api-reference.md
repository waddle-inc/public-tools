## 前提

| 項目         | 内容                                       |
| ------------ | ------------------------------------------ |
| ベース URL   | `/auth`                                    |
| データ形式   | JSON                                       |
| 文字コード   | UTF-8                                      |
| Content-Type | `application/json`                         |
| 利用者       | SSO 連携を行う外部アプリケーションの開発者 |

> **備考**: 表中のパスはベース URL を含む表記です。API サーバーのホスト名・ポートは利用環境に応じて読み替えてください。

---

## 外部アプリが扱う API

| メソッド | パス                    | 呼び出し元                       | 概要                         |
| -------- | ----------------------- | -------------------------------- | ---------------------------- |
| `POST`   | `/auth/exchange/verify` | 外部アプリのバックエンド         | 認可コード検証・トークン発行 |
| `POST`   | `/auth/exchange`        | 認証 WEB（外部アプリは直接不要） | 認可コード発行               |

外部アプリが直接呼び出す必要があるのは、原則として `/auth/exchange/verify` のみです。`/auth/exchange` は認証 WEB がログイン後またはサイレントリフレッシュ時に呼び出し、外部アプリへ渡す認可コードを発行します。

---

## トークンとコード

| 名前                           | 用途                                   | 有効期限                                                     | 外部アプリでの扱い                               |
| ------------------------------ | -------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------ |
| SSO 認可コード                 | 外部アプリ向けアクセストークンとの交換 | 60 秒                                                        | コールバック URL または `postMessage` で受け取る |
| 外部アプリ向けアクセストークン | 外部アプリ側 API の認証                | `JWT_ACCESS_EXPIRES_IN` の設定値（開発環境の既定値は 15 分） | `/auth/exchange/verify` のレスポンスで受け取る   |
| リフレッシュトークン           | 認証システム側セッションの維持         | `JWT_REFRESH_EXPIRES_IN` の設定値（開発環境の既定値は 7 日） | 外部アプリには発行されない                       |

> **注意**: SSO 認可コードは 1 回限り使用できます。期限切れまたは使用済みのコードは `/auth/exchange/verify` で `401 Unauthorized` になります。

## 共通エラーレスポンス

エラーレスポンスは基本的に以下の形式です。`message` はエラー種別により文字列または文字列配列になります。

```json
{
  "statusCode": 400,
  "message": "エラーメッセージ",
  "error": "Bad Request"
}
```

バリデーションエラーが複数ある場合、`message` は配列になります。

```json
{
  "statusCode": 400,
  "message": ["code should not be empty", "code must be a string"],
  "error": "Bad Request"
}
```

## POST /auth/exchange/verify

認可コードを検証し、外部アプリ向けのアクセストークンとユーザー情報を発行します。外部アプリのバックエンドからのサーバー間通信を想定しています。

> **重要**: この API はブラウザから直接呼び出さず、外部アプリのバックエンドから呼び出してください。

### リクエスト

| 項目 | 内容                   |
| ---- | ---------------------- |
| 認証 | 不要（サーバー間通信） |

```json
{
  "code": "<認可コード>"
}
```

| フィールド | 型       | 必須 | バリデーション       | 説明       |
| ---------- | -------- | ---- | -------------------- | ---------- |
| `code`     | `string` | ✓    | 文字列、空でないこと | 認可コード |

### レスポンス

| ステータス                  | 説明                             |
| --------------------------- | -------------------------------- |
| `200 OK`                    | 認可コード検証成功・トークン発行 |
| `400 Bad Request`           | リクエストボディ不正             |
| `401 Unauthorized`          | 認可コード不正・期限切れ・既使用 |
| `500 Internal Server Error` | サーバーエラー                   |

**200 OK**:

```json
{
  "accessToken": "<アクセストークン>",
  "user": {
    "id": "<ユーザー ID>",
    "email": "<メールアドレス>",
    "roles": ["customer"]
  }
}
```

レスポンスの `accessToken` は外部アプリ向けです。JWT の `aud` は `ALLOWED_CLIENTS` に設定された外部アプリのサービス名になり、`auth` ではありません。

**401 Unauthorized**:

```json
{
  "statusCode": 401,
  "message": "認可コードが無効または期限切れです",
  "error": "Unauthorized"
}
```

## POST /auth/exchange

認証済みユーザーに対して SSO 認可コードを発行します。この API は認証 WEB が内部で利用します。外部アプリは通常、この API を直接呼び出しません。

### リクエスト

| 項目 | 内容            |
| ---- | --------------- |
| 認証 | Bearer トークン |

```text
Authorization: Bearer <認証システム向けアクセストークン>
```

```json
{
  "redirect": "https://your-app.example.com/callback"
}
```

| フィールド | 型       | 必須 | バリデーション                                           | 説明                                   |
| ---------- | -------- | ---- | -------------------------------------------------------- | -------------------------------------- |
| `redirect` | `string` | ✓    | 有効な URL 形式、オリジンが `ALLOWED_CLIENTS` に登録済み | 遷移先のコールバック URL（完全な URL） |

### レスポンス

| ステータス                  | 説明                                                   |
| --------------------------- | ------------------------------------------------------ |
| `200 OK`                    | 認可コード発行成功                                     |
| `400 Bad Request`           | `redirect` のオリジンが `ALLOWED_CLIENTS` に含まれない |
| `401 Unauthorized`          | アクセストークン不正、または `aud` が `auth` 以外      |
| `500 Internal Server Error` | サーバーエラー                                         |

**200 OK**:

```json
{
  "code": "<認可コード>"
}
```

---

## `ALLOWED_CLIENTS`

SSO 連携では、認証 API 側で外部アプリのオリジンと発行先 `aud` を `ALLOWED_CLIENTS` に登録しておく必要があります。

```json
[
  {
    "origin": "https://your-app.example.com",
    "aud": "your-app"
  }
]
```

| 項目     | 内容                                                                |
| -------- | ------------------------------------------------------------------- |
| `origin` | `/auth/exchange` の `redirect` から抽出したオリジンと完全一致で照合 |
| `aud`    | `/auth/exchange/verify` で発行するアクセストークンの受信者          |

クライアント側から `aud` を指定することはできません。`aud` はサーバー側で `ALLOWED_CLIENTS` から決定されます。

---

## 実装上の注意点

- `/auth/exchange/verify` は外部アプリのバックエンドから呼び出してください。
- 認可コードの有効期限は 60 秒です。受け取ったら速やかに検証してください。
- 認可コードは 1 回限り使用できます。リトライが必要な場合は認証 WEB から新しい認可コードを取得してください。
- `/auth/exchange/verify` はリフレッシュトークン Cookie を発行しません。
- 外部アプリ向けアクセストークンの更新は、認証 WEB のサイレントリフレッシュフローで新しい認可コードを取得して行います。
