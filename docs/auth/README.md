# SSO 連携公開ドキュメント

このドキュメントは、認証システムの SSO 連携を利用する外部アプリケーションの開発者向けに提供する公開ドキュメントです。

> **閲覧環境について**
> Mermaid 図は [Mermaid 対応エディタ](https://mermaid.js.org/)（GitHub、Obsidian 等）で正しくレンダリングされます。

---

## 概要

外部アプリは認証システムのセッションを利用してユーザー認証を行い、認可コードを介して外部アプリ向けのアクセストークンを取得します。

| 対象                         | 内容                                                                |
| ---------------------------- | ------------------------------------------------------------------- |
| 未認証ユーザーのログイン     | 認証 WEB へリダイレクトし、ログイン後に認可コードを受け取る         |
| 認証済みユーザーの再認証     | サイレントリフレッシュで新しい認可コードを取得する                  |
| 認可コード検証               | 外部アプリのバックエンドから認証 API へ認可コードを送信して検証する |
| 外部アプリ向けトークンの利用 | `aud` が外部アプリ向けのアクセストークンを利用する                  |

> **備考**: アカウント登録、ログイン、パスワード管理などは認証 WEB が内部で提供する機能です。外部アプリ向けの公開ドキュメントでは SSO 連携に必要な仕様のみを扱います。

---

## 前提条件

| 項目           | 内容               |
| -------------- | ------------------ |
| 通信プロトコル | HTTPS              |
| データ形式     | JSON               |
| 文字コード     | UTF-8              |
| Content-Type   | `application/json` |

---

## 用語集

| 用語                           | 説明                                                                                                       |
| ------------------------------ | ---------------------------------------------------------------------------------------------------------- |
| SSO 認可コード                 | 外部アプリ向けアクセストークンと交換するための一時コード。有効期限は 60 秒                                 |
| 外部アプリ向けアクセストークン | 外部アプリ側 API の認証に使用する JWT。`aud` は `ALLOWED_CLIENTS` に設定された外部アプリのサービス名になる |
| リフレッシュトークン           | 認証システム側セッションの維持に使用する HttpOnly Cookie。外部アプリには発行されない                       |
| `ALLOWED_CLIENTS`              | SSO を許可する外部アプリのオリジンと `aud` を定義する認証 API 側の設定                                     |
